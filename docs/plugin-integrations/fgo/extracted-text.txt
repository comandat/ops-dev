FACTURA  EMITERE  POST   Request  Link :   ROOT /factura/emitere  Limitare 1 request la 1 secunda  Timpul maxim necesar pe care FGO il are la dispozitie de la primirea request - ului pana la emiterea unei facturi  este de 15 secunde. In cazul in care acest timp este depasit, prin API ar trebui sa primiti urmatorul mesaj:  { Success = false, Message = "Timpul maxim pentru emiterea unei facturi a expirat. Factura nu a fost emisa si este  necesar sa reincercati. [x]" }  [x] reprezinta un cod intern ce poate avea valori de la 1 la   7   si va ajuta in investigarea posibilelor cauze in cazul in  care este necesar acest lucru.  Timpul de 15 secunde include doar intervalul intre momentul primirii request - ului de emitere de catre FGO, pana  la momentul formarii raspunsului. Timpul total petrecut, in care platforma dvs. formeaza si trimite request - ul  respectiv primeste raspunsul de l a FGO poate fi mai mare decat cele 15 secunde.  Recomandăm ca fiecare apel API să fie executat dupa finalizarea sau obtinerea răspunsului apelului anterior. În  caz contrar, există riscul aparitiei de intarzieri sau blocaje la nivelul companiei Dv, depasindu - se capacitatea  alocata thread - urilor pentru ap leurile care va sunt asignate/disponibile. In conditii atipice, generate de efectuarea  apelurilor API derulate in paralel, care se suprapun totodata cu accesul web sau mobile al utilizatorilor pentru  compania pentru care efectuati apeluri, pot rezulta in m od determinist efecte ca latență crescută pe compania dvs  sau chiar deadlock - uri, deoarece structura de date va trebui să gestioneze simultan mai multe requesturi  incomplete sau concurente.
